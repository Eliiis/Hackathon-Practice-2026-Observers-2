# TailGate

Прогнозирование дохода клиента банка. Оценка качества — взвешенная средняя абсолютная
ошибка. Обучение и вывод выполняются на центральном процессоре; полный путь от исходных
файлов до готового ответа занимает около шести минут.

## 1. Постановка

Оценка качества определена как

```
WMAE = Σ wᵢ·|yᵢ − ŷᵢ| / Σ wᵢ
```

где вес наблюдения является функцией самой целевой переменной:

```
w = min( |y − 84017.07996115311| / 84017.07996115311 ,  2.5707025302320754 )
```

Формула восстановлена по столбцу `w` обучающей выборки и совпадает с ним с точностью
1.3·10⁻¹⁵. Верхнее ограничение вступает в силу ровно при `y = 300 000`.

Знаменатель зависит только от истинных значений и потому постоянен. Следовательно,
минимизация оценки эквивалентна поминутной минимизации числителя, а оптимальным
точечным прогнозом для каждого наблюдения служит **взвешенная условная медиана**
распределения `p(y|x)` с весовой функцией `w(y)`.

## 2. Распределение оценки по сегментам

Весовая функция обращается в ноль у клиента с доходом 84 017 и возрастает при удалении
от этой точки в обе стороны. Поэтому вклад наблюдений в итоговую оценку распределён
крайне неравномерно:

| сегмент дохода | доля наблюдений | доля оценки | WMAE сегмента | смещение |
|---|---|---|---|---|
| до 50 000 | 35.5% | 13.2% | 21 450 | +7 157 |
| 50 000 – 84 000 | 31.6% | 5.0% | 23 952 | −4 227 |
| 84 000 – 150 000 | 20.6% | 8.5% | 47 136 | −3 006 |
| 150 000 – 300 000 | 8.6% | 23.7% | 67 032 | −6 646 |
| свыше 300 000 | 3.7% | **49.5%** | 181 891 | −92 289 |

Половина итоговой оценки формируется 3.7% наблюдений. Совокупно на сегменты свыше
150 000 приходится 73% оценки при 12% наблюдений. Это наблюдение определило все
последующие архитектурные решения: улучшения вне верхнего хвоста распределения
практически не влияют на результат.

Проверочный расчёт: безошибочное предсказание одних лишь наблюдений с доходом свыше
миллиона — 0.28% выборки — улучшило бы оценку на 7 120 единиц.

## 3. Архитектура решения

### 3.1. Целевая функция и шкала

Обучение ведётся с функцией потерь `reg:absoluteerror` по преобразованной величине
`log₁₊(y)` с весами, равными весам оценки качества. Такая постановка даёт взвешенную
условную медиану, то есть теоретический оптимум задачи.

Выбор логарифмической шкалы проверен экспериментально. Обучение на исходной шкале, а
также на квадратном и кубическом корне, улучшает точность в верхнем хвосте (WMAE
сегмента свыше 300 000 снижается со 175 084 до 172 176), однако ухудшает общий результат.
Причина в том, что медиана инвариантна относительно монотонных преобразований лишь
теоретически: деревья строятся по знаку градиента, и в различных шкалах получаются
различные разбиения.

### 3.2. Каскадная декомпозиция

Задача разделена на три модели:

- **разделяющий классификатор** оценивает вероятность `p = P(y ≥ 150 000 | x)`;
- **модель верхнего сегмента** обучается на состоятельных клиентах;
- **базовая модель** обучается на остальных.

Итоговый прогноз собирается как выпуклая комбинация `p·(верхняя) + (1−p)·(базовая)`.
Такая форма соответствует разложению условного математического ожидания по полной
группе событий.

Проверены и отвергнуты альтернативы: жёсткое переключение по порогу ухудшает результат
примерно на 1 500 единиц, дополнительное заострение вероятностей — на 130 и более.

Модель верхнего сегмента дополнительно обучается на диапазоне 100 000 – 150 000 с весом
0.3. Без этого усечение выборки снизу приводит к систематическому занижению её прогнозов
вблизи нижней границы применимости.

### 3.3. Калибровка разделяющего классификатора

Вероятности классификатора преобразуются масштабированием логита с коэффициентом,
подобранным на отложенном последнем месяце обучающей выборки.

Подобранное значение коэффициента устойчиво оказывается меньше единицы (0.45 – 0.60).
Это означает, что преобразование не приводит вероятности в соответствие с наблюдаемыми
частотами, а **увеличивает их разброс**: логит растягивается приблизительно вдвое.
Содержательно классификатор недостаточно уверен — его вероятности сжаты к базовой доле,
и растяжение восстанавливает разделяющую способность.

Указанная интерпретация подтверждена экспериментом: методы, действительно приводящие
вероятности в соответствие с частотами — логистическая калибровка и изотоническая
регрессия — ухудшают результат приблизительно на 3 000 единиц. Механизм даёт около
1 289 единиц улучшения, что делает его вторым по значимости после самой каскадной
декомпозиции.

### 3.4. Поправка на смещение распределения меток

Проверочная выборка относится к более поздним месяцам и содержит систематически более
состоятельных клиентов. Доли сегментов в ней восстанавливаются методом оценки по
матрице ошибок: наблюдаемое распределение предсказанных классов связано с истинным
распределением через матрицу условных вероятностей, оценённую на отложенном месяце
обучающей выборки. Ни одна метка проверочной выборки при этом не используется.

Полученные отношения применяются дважды: как множители весов при обучении и как
априорная поправка к вероятностям разделяющего классификатора.

| сегмент | обучение | проверка | отношение |
|---|---|---|---|
| до 50 000 | 34.99% | 32.05% | 0.916 |
| 50 000 – 84 000 | 31.77% | 32.17% | 1.012 |
| 84 000 – 150 000 | 21.11% | 22.17% | 1.050 |
| 150 000 – 300 000 | 8.54% | 9.04% | 1.059 |
| свыше 300 000 | 3.59% | 4.57% | 1.274 |

Метод следует применять с осторожностью: матрица условных вероятностей близка к
вырожденной, и оценка чувствительна к качеству вспомогательной модели. При изменении
набора признаков оценка доли верхнего сегмента изменялась в пределах 4.19% – 4.57%.
По этой причине принятые доли зафиксированы в `config.yaml` и являются частью
конфигурации решения. Стадия `shift` пересчитывает оценку на текущих признаках и
печатает расхождение с зафиксированными значениями, не изменяя их.

### 3.5. Объединение источников дохода

В данных содержится четырнадцать столбцов, каждый из которых представляет собой
зашумлённое измерение одной и той же величины — дохода клиента. Столбцы существенно
различаются по надёжности: среднеквадратическое отклонение логарифма отношения истинного
дохода к измеренному составляет от 0.26 у наиболее точного источника до 0.88 у наименее
точного. Кроме того, каждый источник заполнен лишь частично, и множества заполненности
пересекаются не полностью.

Применяется следующая процедура. Для каждого источника `s` на обучающей выборке
оценивается масштабный коэффициент `kₛ = median(y / xₛ)`, приводящий измерение к шкале
целевой переменной, и мера рассеяния `σₛ`. Оценки объединяются взвешиванием по обратной
дисперсии:

```
ŷ_объед = Σ (1/σₛ²)·log₁₊(kₛ·xₛ)  /  Σ (1/σₛ²)
```

где суммирование ведётся по источникам, заполненным у данного наблюдения. Это
статистически корректный способ объединения независимых зашумлённых оценок одной
величины, в отличие от среднего арифметического, которое уравнивает источники с
трёхкратной разницей в точности.

Существенное дополнение: величина `σₛ` не является константой, а **оценивается отдельной
моделью для каждого наблюдения**. Источник, надёжный для наёмного работника, может быть
неинформативен для владельца предприятия, чей доход не проходит через фонд оплаты труда.
Вспомогательные модели предсказывают `|log(y / kₛxₛ)|` по контекстным признакам и
обучаются перекрёстной подгонкой в три блока — без этого оценка рассеяния использовала бы
целевую переменную того же наблюдения, для которого строится признак.

Вклад механизма измерен парным сравнением: 311 единиц на локальной проверке, 282 единицы
на публичной оценке.

### 3.6. Поканальная декомпозиция

Столбцы разделены на пять групп по происхождению данных: официальные сведения о
занятости и выплатах, сведения кредитного бюро, обороты по счетам, транзакционная
активность, поведенческие показатели. На каждой группе обучается отдельная модель,
дающая независимую оценку дохода по данным только этого источника.

В каскад передаются: оценки каждой группы, доля заполненности каждой группы, а также
показатели расхождения между оценками — среднеквадратическое отклонение, размах, среднее
и максимум. Содержательно информативно именно расхождение: ситуация, в которой
официальные сведения отсутствуют, а транзакционная активность соответствует высокому
доходу, является самостоятельным признаком. Ансамбль деревьев не способен построить
такой признак самостоятельно, поскольку операция деления одного признака на другой ему
недоступна.

Для обучающей выборки оценки вычисляются по схеме «исключён один месяц»: модель,
предсказывающая наблюдения месяца `m`, обучается на всех прочих месяцах. Для проверочной
выборки используются модели, обученные на всей обучающей выборке.

Вклад механизма: 412 единиц на локальной проверке, 661 на публичной оценке —
наибольший среди всех рассмотренных.

### 3.7. Смешивание с одиночной моделью

Итоговый ответ представляет собой линейную комбинацию `0.7·(каскад) + 0.3·(одиночная
модель)`, обученную на тех же признаках и весах. Пропорция найдена перебором с парным
сравнением; оптимум пологий в диапазоне 0.20 – 0.30.

Смешивание с моделями иных семейств проверено и отвергнуто. Причина неудачи установлена:
выигрыш от объединения возникает при сопоставимом качестве и низкой корреляции
предсказаний, тогда как альтернативные реализации градиентного бустинга оказались либо
существенно слабее (на 9% при коэффициенте корреляции 0.958), либо практически
неотличимы (коэффициент корреляции 0.991).

## 4. Методология проверки

Применяется исключительно проверка с соблюдением хронологии: модель обучается на ранних
месяцах и оценивается на последующих. Обычная перекрёстная проверка со случайным
разбиением на задаче с временной структурой даёт смещённую оценку.

Каждое решение принималось по результатам **парного сравнения с бутстрэп-оценкой
дисперсии** на фиксированном множестве наблюдений. Собственный шум оценки качества
измерен и составляет около 762 единиц при объёме проверочной выборки. Изменения, дающие
менее двух стандартных отклонений улучшения, отклонялись.

Результаты полной проверки на пяти хронологических блоках:

| блок | объём обучения | WMAE | WMAE с поправкой |
|---|---|---|---|
| 1 | 7 243 | 71 291 | 78 157 |
| 2 | 16 108 | 62 159 | 68 738 |
| 3 | 29 521 | 60 135 | 66 751 |
| 4 | 44 379 | 57 701 | 63 857 |
| 5 | 60 572 | 59 270 | 65 652 |
| среднее | | **62 111** | **68 631** |

Ухудшение на пятом блоке при возросшем объёме обучения отражает не регресс модели, а
различие сложности месяцев. Это иллюстрирует, почему абсолютные значения отдельных
измерений не могут служить основанием для решений.

**Абсолютные значения локальной проверки несопоставимы с публичной оценкой**, и это не
является дефектом методологии. Точка нулевого веса (84 017) и верхнее ограничение
(300 000) заданы в номинальных денежных единицах, тогда как доходы за полгода между
выборками возросли приблизительно на 10%. Прямое моделирование показывает, что при
полностью неизменной модели одно только это обстоятельство увеличивает оценку с 59 126
до 67 116. Сопоставлять допустимо только парные разности на одних и тех же наблюдениях.

## 7. Особенности данных

- Разделитель полей — точка с запятой, десятичный разделитель — запятая. Тип столбца
  определяется по доле значений, допускающих числовую интерпретацию.
- Дата в пределах месяца принимает единственное значение — последний день месяца.
  Столбец даты является меткой отчётного периода, а не датой события.
- Столбец `first_salary_income` заполнен у 11.3% обучающей выборки и у 0% проверочной;
  исключён.
- Одиннадцать счётчиков вида «дней с момента события» исключены: они возрастают строго
  по календарю. Для `dp_ils_days_from_last_doc` прирост составляет 30.3 в месяц при
  коэффициенте детерминации 1.00.
- Проверка на утечку целевой переменной выполнена по семи направлениям: пересечение
  идентификаторов между выборками отсутствует, повторяющиеся клиенты отсутствуют, ни один
  признак не воспроизводит целевую переменную, поканальные оценки на обучающей выборке
  не точнее, чем на честной проверке.

## 8. Запуск

```bash
uv sync
uv run snakemake all               # построение ответа
uv run snakemake validate report   # проверка и аналитический отчёт
uv run snakemake shift             # переоценка смещения и сверка с config.yaml
uv run snakemake -n                # план выполнения без запуска
```

Исходные файлы размещаются в каталоге `data/raw/` под именами `train.csv`, `test.csv`,
`sample_submission.csv`. Результат появляется в `submissions/submission.csv`,
аналитический отчёт — в `reports/analysis.md`.

## 9. Воспроизводимость

Все источники случайности зафиксированы: модели используют начальное значение
генератора случайных чисел из `config.yaml`, разбиения выполняются по календарю,
принятые доли сегментов заданы в конфигурации в полной точности. Скрытого состояния
вне репозитория нет: каталог `artifacts/` содержит только производные файлы и в
построении ответа не участвует.

Полный запуск из чистой копии репозитория проверен: сборка признаков воспроизводится
побайтово, переоценка долей на одинаковых признаках возвращает одинаковые значения,
итоговый `submissions/submission.csv` совпадает с сохранённым файлом с точностью до
байта. Сохранённый файл соответствует конфигурации, получившей публичную оценку
75 910.88.

Единственная оговорка касается переноса на другое оборудование: деревья обучаются в
несколько потоков, порядок суммирования чисел с плавающей запятой зависит от числа
ядер, поэтому на иной машине отдельные прогнозы могут отличаться в младших разрядах.
Расхождение не носит систематического характера и на итоговую оценку практически не
влияет.
